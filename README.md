# pet_python_bot — управление виндой через ботика, развернутого на WSL/Linux

Архитектура из двух компонентов (Docker-контейнер для бота в WSL/Linux и нативный агент для хостовой Windows), позволяющая удаленно управлять системой, отправлять команды в PowerShell и выполнять резервное копирование.

## Доступные команды
* `/start` — проверка доступности бота и сети.
* `/backup` — создание сжатого tar-архива домашней директории WSL, сохранение копии на хосте и отправка архива в Telegram.
* `/scr` — запрос снимка экрана у Windows-агента и отправка изображения в чат.
* `/say текст` — отправка текстового пакета на Windows-агент для озвучки через аудиовыход хоста.
* скоро будут новые!

## Структура проекта
* `.github/workflows/` — конфигурация CI/CD пайплайна GitHub Actions.
* `win_agent/` — исходный код агента, batch-скрипты и VBS-файлы для фонового запуска на Windows.
* `wsl_bot/` — логика работы Telegram-бота для Linux/WSL окружения.
* `Dockerfile` — многоэтапная сборка окружения бота.
* `docker-compose.yml` — манифест для развертывания инфраструктуры.

## Быстрый старт

### 1. Настройка Windows агента (Хост)
1. Перейдите в директорию `win_agent` и запустите VBS-скрипт. Он подымет HTTP-сервер на порту 5000 в фоновом режиме без открытия окошка консоли(питон нужен):
```cmd
start_hidden.vbs
```

### 2. Настройка бота (WSL / Linux)
1. В корневом каталоге проекта создайте файл `.env` на основе примера `.env.example` и заполните переменные окружения. Переменная PROXY нужна для обхода блокировок Telegram API:
```env
TELEGRAM_TOKEN=your_bot_token_here
TELEGRAM_CHAT_ID=your_telegram_chat_id
BACKUP_WIN_PATH=/path/to/windows/backup/folder
TELEGRAM_PROXY=<your proxy>
```

2. Запустите контейнеризованное приложение. Docker Compose автоматически соберет образ из Dockerfile, учтет изменения в wsl_bot и запустит процесс в фоне:
```bash
docker compose up -d --build
```

3. Просмотр логов бота в реальном времени для верификации успешного соединения с Telegram API:
```bash
docker compose logs -f
```
